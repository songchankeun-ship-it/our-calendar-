# 결혼준비 워크스페이스 v2 — 리디자인 기획서

> "업체 비교 + 계약 관리 + 예산 통제"에 강한 실행형 워크스페이스

---

## 1. 최종 IA (Information Architecture)

```
📱 앱
│
├── 🏠 홈 (Command Center)
│   ├── 🔴 지금 해야 할 것 1개 (Hero Action Card)
│   ├── ⚠️ 알림 모음 (잔금 임박 / 예산 초과 / 마감 임박)
│   ├── 📊 진행률 요약 (체크리스트 / 예산 / 업체)
│   ├── 📋 이번 주 할 일 (3개 미리보기)
│   ├── 🏢 업체 현황 (파이프라인 미니맵)
│   ├── 🕐 최근 활동 (2~3개 인라인)
│   └── 👤 파트너 상태 (마지막 활동)
│
├── 🏢 업체 (Vendor CRM)
│   ├── [탭] 목록 뷰 — 카드 리스트 + 필터
│   ├── [탭] 보드 뷰 — 칸반 (상태별 컬럼)
│   ├── [탭] 비교 뷰 — 비교 테이블
│   ├── 업체 상세
│   │   ├── 상태 파이프라인
│   │   ├── 기본 정보 + 연락처
│   │   ├── 💰 견적/계약 (예산 연동)
│   │   ├── ✅ 장점 / ❌ 단점 / ⚠️ 우려
│   │   ├── ⭐ 내 평가 + 파트너 평가
│   │   ├── 📎 관련 문서
│   │   ├── 📋 연결된 체크리스트
│   │   ├── 🎯 다음 액션 (CTA)
│   │   └── 🕐 변경 이력 (인라인)
│   └── 업체 추가/편집
│
├── 💰 예산 (Budget Control)
│   ├── 총괄 대시보드
│   │   ├── 목표 / 확정 / 납부 / 잔여
│   │   ├── 카테고리별 진행률
│   │   └── ⚠️ 초과 위험 표시
│   ├── 카테고리 상세
│   │   └── 항목별 계약금/잔금/추가금/납부일
│   └── 💳 결제 일정 (잔금 캘린더)
│
├── ✅ 일정 (Timeline + Checklist)
│   ├── [탭] 타임라인 뷰 — 월별 마일스톤
│   ├── [탭] 체크리스트 뷰 — 전체 할 일 리스트
│   ├── 필터: 상태 / 담당자 / 마감 임박 / 카테고리
│   └── 각 항목 → 업체/문서/예산 연결
│
└── 📎 문서함 (Document Vault)
    ├── 전체 문서 (최근순)
    ├── 필터: 유형 / 업체 / 카테고리
    ├── 유형: 계약서 / 견적서 / 상담캡처 / 이미지 / 메모
    └── 업체별 그룹 뷰
```

## 2. 화면별 목적 + 핵심 컴포넌트

### Screen 1: 온보딩 (3단계)

**목적:** 30초 안에 맞춤 워크스페이스 자동 생성

| 단계 | 입력 | 자동 생성 결과 |
|-----|------|-------------|
| Step 1: 기본 | 예식 예정일, 지역, 이름 | D-day 카운터 |
| Step 2: 규모 | 하객 수, 준비 단계, 웨딩홀 계약 여부 | 맞춤 체크리스트 (단계별 필터링) |
| Step 3: 예산+파트너 | 총 예산, 파트너 초대 | 카테고리별 예산 자동 분배, 룸 생성 |

**핵심 컴포넌트:**
- 단계 인디케이터 (● ● ○)
- 선택형 칩 (지역, 준비 단계)
- 예산 자동 분배 프리뷰 ("웨딩홀 40% / 스드메 10% / ...")
- "파트너 초대" or "나중에" 분기

---

### Screen 2: 홈 (Command Center)

**목적:** "지금 뭘 해야 하는지" 1초 안에 파악

**삭제:** 활동 로그 탭 (여기에 인라인 2~3개로 축소)

**핵심 컴포넌트:**

| 컴포넌트 | 역할 | 우선순위 |
|---------|------|--------|
| **Hero Action Card** | "지금 가장 먼저 해야 할 1개" — 탭하면 해당 화면으로 이동 | ★★★★★ |
| **Alert Strip** | 잔금 D-7 / 예산 초과 / 상담 내일 — 최대 3개 | ★★★★★ |
| **Progress Ring** | 체크리스트 완료율 (원형 게이지) | ★★★★☆ |
| **Stat Cards** | 예산 잔여 / 업체 현황 / 체크리스트 — 2x2 그리드 | ★★★★☆ |
| **이번 주 할 일** | 최대 3개, 체크 가능, "전체 보기" 링크 | ★★★★☆ |
| **업체 미니맵** | 칸반 축약 (관심 3 → 상담 2 → 계약 1) 가로 바 | ★★★☆☆ |
| **최근 활동** | 파트너 활동 2~3개 인라인. 독립 탭 아님 | ★★★☆☆ |

---

### Screen 3: 업체 (Vendor CRM) — 3가지 뷰

**목적:** 업체를 찾고 → 비교하고 → 계약까지 한 화면에서

**핵심 변경:** 비교 기능이 메인 탭 레벨로 승격

#### 3-1. 목록 뷰
- 카드 리스트
- 상단: 카테고리 필터 칩 (전체 / 웨딩홀 / 스드메 / 예복 / 스냅)
- 카드: 업체명, 카테고리, 상태 배지, 견적, 별점, "비교 담기" 체크박스
- FAB: + 업체 추가

#### 3-2. 보드 뷰 (칸반)
- 가로 스크롤 컬럼: 관심 → 상담예정 → 상담완료 → 견적 → 비교중 → 계약
- 각 컬럼에 업체 미니카드 (이름 + 견적)
- 드래그 or 탭으로 상태 이동

#### 3-3. 비교 뷰 (비교표)
- "비교 담기"한 업체들을 테이블로 비교
- 행: 가격 / 추가금 / 날짜 / 포함사항 / 응대 만족도 / 내 평가 / 파트너 평가
- 하단: "최종 선택" 버튼
- 카테고리별 비교 (웨딩홀끼리, 스드메끼리)

---

### Screen 4: 업체 상세 (강화)

**목적:** 이 업체에 대한 모든 정보 + "다음에 뭘 해야 하지?"

**핵심 컴포넌트:**

| 컴포넌트 | 내용 |
|---------|------|
| **상태 파이프라인** | 7단계 프로그레스 바 + 현재 위치 |
| **다음 액션 CTA** | "상담 예약하기" / "견적 요청하기" / "계약금 납부" — 상태에 따라 자동 변경 |
| **기본 정보** | 업체명, 카테고리, 담당자, 연락처, 상담일 |
| **견적/계약** | 견적 금액, 계약금, 잔금, 추가금 → 예산 탭과 연동 |
| **장단점** | ✅ 장점 / ❌ 단점 / ⚠️ 우려 — 자유 입력 |
| **내 평가** | 5점 척도 (가격/응대/품질/위치/분위기) + 한줄 메모 |
| **파트너 평가** | 파트너가 별도로 평가 → 둘 다 볼 수 있음 |
| **관련 문서** | 이 업체의 견적서/계약서/캡처 — 문서함 연결 |
| **연결된 체크리스트** | "웨딩홀 계약" 항목이 이 업체에 연결됨 |
| **변경 이력** | 이 카드의 수정 기록 (인라인, 접히는 형태) |

---

### Screen 5: 업체 비교 (신규)

**목적:** 같은 카테고리 업체를 나란히 비교해서 최종 결정

**웨딩홀 비교 예시:**

| 항목 | 더클래식500 | 라브르 | 그랜드하얏트 |
|-----|-----------|-------|-----------|
| 견적 | 1,200만 | 1,400만 | 2,200만 |
| 추가금 | +50만 | +120만 | +200만 |
| 보증인원 | 120명 | 100명 | 150명 |
| 식대 | 7만 | 8만 | 12만 |
| 날짜 | ✅ 토 점심 | ✅ 토 저녁 | ❌ 없음 |
| 주차 | 무료발렛 | 유료 | 무료 |
| 내 평가 | ⭐ 4.5 | ⭐ 4.0 | ⭐ 3.5 |
| 파트너 | ⭐ 4.0 | ⭐ 4.5 | ⭐ 3.0 |

하단: [최종 선택] 버튼 → 선택 시 상태 자동 변경 + 예산 연동

---

### Screen 6: 예산 (Budget Control)

**목적:** 돈이 어디서 새는지 한눈에. 앞으로 얼마가 더 나갈지.

**핵심 구조 변경:**

```
기존: 총예산 / 확정 / 잔여 (3개)
변경: 목표 / 확정 / 납부 / 잔여 / 앞으로 나갈 돈 (5개)
```

**핵심 컴포넌트:**

| 컴포넌트 | 내용 |
|---------|------|
| **4-Box Summary** | 목표 예산 / 확정 총액 / 이미 낸 돈 / 앞으로 나갈 돈 |
| **경고 배너** | "스드메 50만 초과" / "이번 달 잔금 900만 예정" |
| **카테고리 리스트** | 각각 진행률 바 + 확정/계획 + 초과 여부 |
| **결제 일정** | 다가오는 잔금/결제 시간순 리스트 (캘린더 아닌 리스트) |
| **카테고리 상세** | 탭 시 → 항목별 계약금/잔금/추가금/납부일/상태 |

---

### Screen 7: 일정 (Timeline + Checklist)

**목적:** "언제까지 뭘 해야 하지?"를 두 가지 관점으로

**2가지 뷰:**

| 뷰 | 용도 |
|---|------|
| **타임라인** | 월별 마일스톤. "12개월 전 → 6개월 전 → D-day" 흐름 |
| **체크리스트** | 전체 할 일 플랫 리스트. 필터로 빠르게 탐색 |

**필터:**
- 상태: 전체 / 미완료 / 진행중 / 완료
- 담당자: 전체 / 신부 / 신랑 / 둘 다
- 마감: 임박순 (D-7 이내 강조)
- 카테고리: 웨딩홀 / 스드메 / 예복 / ...

**연결:** 각 항목에서 → 관련 업체 / 관련 문서 / 예산 항목으로 이동 가능

---

### Screen 8: 문서함 (Document Vault)

**목적:** 카톡 캡처, 견적서, 계약서가 사라지지 않는 곳

**핵심 컴포넌트:**

| 컴포넌트 | 내용 |
|---------|------|
| **업로드 FAB** | 사진/파일 업로드 |
| **필터 칩** | 유형: 전체/계약서/견적서/상담캡처/이미지/메모 |
| **업체별 그룹** | 접기/펼치기. "더클래식500 (3)" |
| **문서 카드** | 썸네일 + 제목 + 유형 배지 + 업체 + 날짜 |
| **검색** | 문서 제목 / 업체명 검색 |
| **빈 상태** | "견적서를 받으면 여기에 보관하세요" |

## 3. UX 개선안

### 원칙: "실행형 워크스페이스"로 느껴지게 하는 5가지

1. **모든 화면에 "다음 액션"이 있다**
   - 홈: Hero Action Card
   - 업체 상세: "상담 예약하기" CTA
   - 예산: "잔금 납부 기록" CTA
   - 체크리스트: 체크 가능한 첫 번째 항목

2. **데이터는 한 번 입력하면 여러 곳에 반영된다**
   - 업체에 견적 입력 → 예산에 자동 반영
   - 업체 상태를 "계약"으로 변경 → 체크리스트 자동 완료
   - 문서를 업체에 첨부 → 문서함에도 자동 등록

3. **경고가 먼저 보인다**
   - 예산 초과: 빨간색 표시 + 홈 알림
   - 잔금 임박: 홈 상단 + 예산 탭 상단
   - 체크리스트 마감: 홈 할 일 + 일정 탭 필터

4. **둘이 함께 쓰되, 혼자 써도 완전히 동작한다**
   - 파트너 없이도 모든 기능 사용 가능
   - 파트너가 있으면: 활동 로그, 담당자, 이중 평가가 보너스
   - 강제 초대 없음

5. **톤은 감성이 아닌 신뢰**
   - 블랙 + 화이트 + 워밍그레이
   - 이모지 최소화 (기능 아이콘으로만)
   - 큰 숫자 + 명확한 상태 + 깔끔한 여백

## 4. 삭제할 요소

| 삭제 | 이유 | 대체 |
|-----|------|------|
| **활동 로그 독립 탭** | 메인 탭 5개에 비해 중요도 낮음 | 홈에 인라인 2~3개 + 각 상세 화면의 변경 이력 |
| **홈의 과도한 stat cards** | 숫자만 나열하면 행동 유도 안 됨 | Hero Action Card + Alert Strip이 대체 |
| **단순 체크리스트 온보딩** | "체크리스트만" 생성은 약함 | 맞춤 체크리스트 + 예산 분배 + 업체판 동시 생성 |
| **업체 카드의 긴 메모** | 목록에서 메모가 길면 스캔 불가 | 상세에서만 표시. 목록은 1줄 요약 |

## 5. 추가할 핵심 화면

| 신규 | 이유 |
|-----|------|
| **업체 비교표** | 결혼준비의 핵심 의사결정. 웨딩홀 3곳을 나란히 비교해야 계약을 결정함 |
| **업체 보드 뷰 (칸반)** | 상태 관리의 직관성. "지금 어떤 업체가 어떤 단계에 있는지" 한눈에 |
| **결제 일정 (잔금 캘린더)** | 예산 탭 내. 앞으로 나갈 돈을 시간순으로. 현금 흐름 파악 |
| **업체 상세 — 장단점/평가** | 비교를 위한 구조화된 데이터. 자유 메모가 아닌 항목별 평가 |
| **문서함 메인 탭** | 견적서/계약서는 핵심 자산. 카톡에 묻히지 않게 전용 공간 |

---

## 6. 화면 간 연결 맵

```
업체 등록 ──→ 예산 자동 반영
    │              │
    ▼              ▼
업체 상태 변경 → 체크리스트 자동 완료
    │
    ▼
문서 첨부 ──→ 문서함에 자동 등록
    │
    ▼
비교 담기 ──→ 비교표에 추가
    │
    ▼
최종 선택 ──→ 상태 "계약" + 예산 "확정"
```

모든 데이터는 **업체 카드를 중심으로 연결**된다.
업체 카드가 이 앱의 핵심 오브젝트.
